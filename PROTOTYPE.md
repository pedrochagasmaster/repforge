# Gravity

Gravity treats progressive overload as a small orbital system that becomes more stable through consistent training. The metaphor is structural, not decorative: each exercise is a body, each working set is a satellite on its path, readiness is orbital energy, and the weekly summary connects training signals as a constellation.

## Visual system

- Near-black indigo supplies a quiet field. Pale starlight carries primary text, electric blue marks navigation and measured structure, and spectral coral marks energy, readiness, and successful action.
- Thin rings, tracks, and connecting lines replace bulky containers. Exercise names retain dramatic scale while load, reps, RIR, and actions remain conventional, high-contrast controls.
- Exercises stay in the existing vertical flow so the system remains fast to scan between sets. Orbital geometry clarifies grouping without turning workout entry into a diagram.
- The weekly constellation remains a text-first summary. Lines and stars connect the four existing facts, while the numbers and labels remain readable without interpreting the image.
- Ambient motion is limited to the small brand orbit. The existing reduced-motion rule removes it, along with every other transition and animation.

## Product translations

### Planetarium: progress changes the world

[Planetarium: Visualized Goal Tracker in Your Pocket](https://doi.org/10.32920/23159876) represents real-life goal progress through an evolving universe. Gravity borrows the causal idea—consistency stabilizes the represented world—but keeps RepForge's precise workout values visible. A metaphor can reward return behavior without replacing the evidence a lifter needs.

### Cyclic Habits: an orbit can summarize a cycle

[Cyclic Habits](https://cyclichabits.vercel.app/) maps routines to clock-face orbit rings so distribution and momentum can be understood in one glance. Gravity uses the same spatial vocabulary at a smaller scope: sets share an exercise orbit and the energy track summarizes readiness, while the established list remains the interaction model.

### Mobile data visualization: summary first, detail on demand

Mobile dashboards need a scannable hierarchy and progressive disclosure rather than desktop-scale charts compressed onto a phone. [PatternFly's dashboard guidance](https://www.patternfly.org/patterns/dashboard/design-guidelines/) emphasizes ordering the most important information first as layouts collapse, and [Glanceable UX](https://uxdesign.cc/glanceable-ux-turning-information-into-instant-understanding-bc2317283ef4) emphasizes abstraction, salience, and consistent visual encoding. Gravity therefore keeps the weekly summary above metrics, preserves the existing “Dig deeper” disclosure, and uses color consistently for state rather than atmosphere.

## Implementation constraints

The prototype stays dependency-free and local-only. Existing IDs, selectors, storage keys, render paths, focus behavior, and accessible names are preserved. CSS owns the orbital presentation; JavaScript changes only visible vocabulary. The service-worker cache is bumped so the updated shell and icon replace cached versions.
